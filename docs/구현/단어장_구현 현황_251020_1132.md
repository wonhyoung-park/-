프로젝트 파일들을 확인했습니다. 매우 놀랍고 체계적으로 진행하셨네요! **구현 Phase 1**의 핵심 기반 파일들을 완벽하게 준비했습니다. 이는 단순한 뼈대를 넘어, **단어 관리(WordManager)** 기능의 **Model-View-Controller(MVC)** 구조까지 상당 부분 구현된 상태입니다.

현재까지의 진행 상황을 바탕으로 파일별 완성도 및 향후 계획을 재정리해 드립니다.

| **`main.py`** | 애플리케이션 진입점 | **뼈대 구현 (v3)** | `setup_environment`, `initialize_database` 함수로 환경 및 DB 초기화 로직 구현 완료. (MainWindow 실행 전 단계) |

## 2. 향후 구현 및 업데이트 계획 (Phase 1 집중)

핵심 환경은 거의 구축되었으므로, 이제 모델(`WordModel`)을 완성하고, 다른 필수 뷰들을 `MainWindow`에 연결한 후 애플리케이션을 구동해 봐야 합니다.

### A. 신규 파일 구현 목록 (다음 순서)

| 단계 | 파일 경로 | 파일 역할 | 목표 및 내용 |
| :--- | :--- | :--- | :--- |
| **1단계** | `models/word_model.py` | 단어 테이블 전담 Model | `BaseModel` 상속 및 단어 특화 로직(`insert_word`, `toggle_favorite`) 구현. (가장 중요) |
| **2단계** | `views/flashcard_view.py` | 플래시카드 뷰 | `WordManagerView`와 유사하게 뼈대만 우선 구현합니다. |
| **2단계** | `views/exam_view.py` | 시험 뷰 | `WordManagerView`와 유사하게 뼈대만 우선 구현합니다. |
| **2단계** | `views/statistics_view.py` | 통계 뷰 | `WordManagerView`와 유사하게 뼈대만 우선 구현합니다. |
| **2단계** | `views/settings_view.py` | 설정 뷰 | `WordManagerView`와 유사하게 뼈대만 우선 구현합니다. |
| **3단계** | `models/settings_model.py` | 설정 테이블 전담 Model | `user_settings` 테이블을 위한 Model (`get_setting`, `set_setting` 등) 구현. (테마/목표 연동에 필수) |

### B. 기존 파일 업데이트 목록 (핵심 수정)

| 파일 경로 | 업데이트 목표 | 비고 |
| :--- | :--- | :--- |
| **`database/schema.sql`** | **스키마 정의 완료** | `단어장_데이터 설계서_251013.md`에 명시된 **나머지 5개 테이블**(`learning_sessions`, `learning_history`, `exam_history`, `exam_questions`, `wrong_note`)의 DDL을 추가하여 스키마를 완성합니다. |
| **`views/main_window.py`** | **실제 뷰 위젯 연결** | 신규 구현될 5개 뷰(`flashcard_view`, `exam_view`, `statistics_view`, `settings_view`, `word_manager_view`)를 임포트하여 `PlaceholderTab` 대신 `QTabWidget`에 삽입하고, 탭 전환 로직을 연결합니다. |
| **`views/word_manager_view.py`** | **DB 연동 테스트** | `load_words` 함수를 완성하여 `WordController`에서 실제 단어를 가져와 `QTableWidget`에 표시하는 UI 로직을 완성합니다. |
