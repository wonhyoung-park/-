## 1. 개요

### 1.1 목적
본 문서는 Smart Vocab Builder의 체계적인 구현을 위해 Phase별 구현 순서, 단계별 작업 내역, 파일 구조를 정의합니다.

### 1.2 구현 원칙
1. **점진적 개발**: MVP(최소 기능 제품) 우선 구현 후 기능 확장
2. **단계별 검증**: 각 Phase 완료 후 테스트 및 검증
3. **모듈별 독립성**: 기능별 모듈 분리로 유지보수 용이
4. **200줄 제한**: 한 번에 200줄 이상 코드는 파일 1개씩 구현

### 1.3 기술 스택
- **GUI**: PyQt5
- **DB**: SQLite3 (Raw SQL)
- **차트**: matplotlib
- **언어**: Python 3.8+
- **패키징**: PyInstaller

## 5. 구현 시 주의사항

### 5.1 코딩 컨벤션
1. **파일 헤더**: 모든 파일 최상단에 작성 날짜, 파일 위치, 목적 명시
   ```python
   # 2025-10-20 - Smart Vocab Builder - 단어 관리 Model
   # 파일 위치: models/word_model.py
   ```

2. **Docstring**: 모든 클래스와 함수에 docstring 작성
   ```python
   def add_word(self, english, korean, memo=None):
       """
       단어 추가
       
       Args:
           english (str): 영어 단어
           korean (str): 한국어 뜻
           memo (str, optional): 메모
           
       Returns:
           int: 추가된 단어 ID
           
       Raises:
           ValueError: 필수 입력값 누락 시
       """
   ```

3. **SQL 쿼리**: 빈 줄 없이 작성, UNION ALL 위아래 여백 제거
   ```python
   query = """
   SELECT word_id, english, korean
   FROM words
   WHERE is_favorite = 1
   ORDER BY english
   """
   ```

4. **에러 처리**: try-except 블록으로 예외 처리, 로깅 포함
   ```python
   try:
       result = self.db.execute_query(query)
   except Exception as e:
       logger.error(f"단어 조회 실패: {e}")
       raise
   ```

### 5.2 데이터베이스 관리
1. **Parameterized Query**: SQL Injection 방지
   ```python
   query = "INSERT INTO words (english, korean) VALUES (?, ?)"
   self.db.execute_update(query, (english, korean))
   ```

2. **트랜잭션**: 여러 작업은 트랜잭션으로 묶기
   ```python
   conn.execute("BEGIN TRANSACTION")
   try:
       # 여러 작업
       conn.execute("COMMIT")
   except:
       conn.execute("ROLLBACK")
   ```

3. **날짜 형식**: ISO 8601 형식 사용
   ```python
   from datetime import datetime
   datetime.now().isoformat()  # "2025-10-20T14:30:00"
   ```

### 5.3 UI 개발
1. **시그널/슬롯**: PyQt5 시그널로 이벤트 처리
   ```python
   self.btn_add.clicked.connect(self.on_add_clicked)
   ```

2. **UI 스레드**: DB 작업은 별도 스레드에서 실행 (추후 최적화)

3. **리소스 경로**: 상대 경로 사용
   ```python
   theme_path = os.path.join("resources", "styles", "light_theme.qss")
   ```

### 5.4 테스트
1. **단위 테스트**: 각 함수별 테스트 케이스 작성
2. **경계값 테스트**: 빈 값, null, 최대값 등 테스트
3. **통합 테스트**: 전체 시나리오 테스트
